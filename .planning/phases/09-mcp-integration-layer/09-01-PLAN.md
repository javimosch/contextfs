<task type="auto">
  <name>Task 1: Create RTK initialization script</name>
  <files>contextfs/scripts/init-rtk.sh</files>
  <action>
    Create `contextfs/scripts/init-rtk.sh` that:
    
    1. Sets RTK binary path to `/usr/local/bin/rtk` (per Phase 8 installation location)
    
    2. Checks `CONTEXTFS_RTK_ENABLED` environment variable:
       - If "false": Set status to "disabled", log "[RTK] Explicitly disabled by CONTEXTFS_RTK_ENABLED=false"
       - If "true" or unset: Proceed to binary detection
       - If invalid value: Log warning "[RTK] Invalid CONTEXTFS_RTK_ENABLED value, treating as unset", proceed to auto-detect
    
    3. Binary detection (when not explicitly disabled):
       - Check if RTK binary exists and is executable at `/usr/local/bin/rtk`
       - Run `rtk --version` to verify functionality
       - Capture version output (e.g., "0.23.0")
       - Set `RTK_STATUS="enabled"` if successful
       - Set `RTK_STATUS="unavailable"` if binary missing or version check fails
    
    4. Logging (verbose startup):
       - Always log: "[RTK] Binary found at /usr/local/bin/rtk" (if exists)
       - Always log: "[RTK] Version: {version}" (if available)
       - Always log: "[RTK] Status: {enabled|disabled|unavailable}"
       - Log format: [RTK] prefix for easy grepping
    
    5. Export for child processes:
       - Export `CONTEXTFS_RTK_STATUS={enabled|disabled|unavailable}`
       - Export `CONTEXTFS_RTK_VERSION={version}` (if detected)
    
    6. Execute main command:
       - Use `exec "$@"` to replace script process with main container process
       - This ensures initialization completes BEFORE MCP server starts
    
    Use `/bin/sh` for maximum compatibility with Alpine Linux. Follow shell scripting best practices from Phase 8 scripts (set -e, proper error handling).
  </action>
  <verify>
    <automated>
      chmod +x contextfs/scripts/init-rtk.sh && \
      /bin/sh -n contextfs/scripts/init-rtk.sh && \
      echo "Script syntax OK"
    </automated>
  </verify>
  <done>
    - Script exists at `contextfs/scripts/init-rtk.sh` with executable permissions
    - Script passes shell syntax check (`sh -n`)
    - Script contains RTK detection logic for enabled/disabled/unavailable states
    - Script logs status with [RTK] prefix
    - Script exports CONTEXTFS_RTK_STATUS environment variable
    - Script ends with `exec "$@"` to run main command
  </done>
</task>

    4. If Dockerfile already has an ENTRYPOINT, update it to chain properly:

    Per Phase 8 decisions: RTK is installed in runtime-full stage at `/usr/local/bin/rtk`
