## Integration Point

bash_script_once in command-runner.js executes scripts via shell. This plan creates an adapter that:
1. Parses individual commands from the script
2. Routes supported commands through RTK
3. Falls back to shell for complex commands (pipes, redirects, conditionals)
4. Combines results preserving exit semantics
</context>

**Measurement Helper:**
```javascript
async function measureReduction(command, args, cwd) {
  const nativeResult = await runNative(command, args, cwd);
  const rtkResult = await runRTK(command, args, cwd);
  const nativeChars = nativeResult.stdout.length;
  const rtkChars = rtkResult.stdout.length;
  const reduction = Math.round(((nativeChars - rtkChars) / nativeChars) * 100);
  return { nativeChars, rtkChars, reduction };
}
```

Create server/mcp/tools/bash-rtk-adapter.js to enable RTK for bash_script_once tool:

**Problem:** bash_script_once runs entire scripts through shell. We need to:
1. Parse individual commands from scripts
2. Route supported simple commands through RTK
3. Keep complex commands (pipes, redirects) in shell

**BashRTKAdapter class:**

parseScript(script):
- Split script into lines
- Parse each line to identify:
  - Simple commands: `ls -la`, `git status`, `grep pattern file`
  - Complex commands: pipes `|`, redirects `>`, conditionals `&&`, `||`, subshells `$()`
- Return array of { type: 'simple'|'complex', command, args, line }

executeScript(script, options):
- Parse script into commands
- For each command:
  - If simple and RTK-supported: use SpawnWrapper
  - If complex or unsupported: collect for shell execution
- Execute simple commands sequentially, tracking exit codes
- Execute complex commands via original shell method
- Combine results preserving exit semantics (first non-zero wins)

shouldUseRTKForLine(line):
- Returns true if:
  - No pipes (|)
  - No redirects (> >> <)
  - No conditionals (&& ||)
  - No command substitution ($())
  - No background processes (&)
  - Command is in RTK allowlist

**Integration with command-runner.js:**
Modify runBashScriptOnce to use BashRTKAdapter when:
- RTK is enabled (CONTEXTFS_RTK_ENABLED !== 'false')
- Script contains at least one simple command
- Not explicitly bypassed with # native: prefix comment

**Example:**
```bash
# Script with mixed commands
ls -la                    # RTK
find . -name "*.js"      # RTK
ls | grep test           # Shell (pipe)
echo "done"              # RTK (cat passthrough)
```
