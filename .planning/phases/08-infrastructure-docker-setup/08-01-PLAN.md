## Key Implementation Details

**Binary URL Pattern:**
```
https://github.com/rtk-ai/rtk/releases/download/v${VERSION}/rtk-${ARCH}-unknown-linux-musl.tar.gz
```

**Architecture Mapping:**
| Docker TARGETARCH | RTK Archive Arch |
|-------------------|------------------|
| amd64 | x86_64 |
| arm64 | aarch64 |

**Pitfalls to Avoid (from RESEARCH):**
1. Use `*-unknown-linux-musl.tar.gz` (not gnu) for Alpine compatibility
2. Use `curl -fsSL` to follow redirects and fail on errors
3. Always `chmod +x` after extraction
4. Use ARG for version to enable layer caching

**Existing Dockerfile Structure:**
- Stage 1: builder (npm ci)
- Stage 2: runtime-base (tini + node + minimal code)
- Stage 3: runtime-full (adds git, ripgrep, tree, curl, bash) ← RTK goes here

  <action>
    Modify the runtime-full stage in contextfs/Dockerfile to install RTK binary:

    1. After the existing `RUN apk add --no-cache` command (after line 57), add:
       - ARG RTK_VERSION=0.23.0
       - ARG TARGETARCH
       - RUN command that:
         a. Maps TARGETARCH to RTK_ARCH (amd64→x86_64, arm64→aarch64)
         b. Downloads from GitHub releases using curl -fsSL
         c. Extracts with tar -xzf to /usr/local/bin
         d. Sets execute permissions with chmod +x
         e. Verifies with `rtk --version`
         f. Cleans up /tmp/rtk.tar.gz

    2. Use this exact pattern from RESEARCH.md:
       ```dockerfile
       # Install RTK binary with multi-architecture support
       ARG RTK_VERSION=0.23.0
       ARG TARGETARCH

       RUN set -eux; \
           case "${TARGETARCH}" in \
               amd64) RTK_ARCH='x86_64' ;; \
               arm64) RTK_ARCH='aarch64' ;; \
               *) echo "Unsupported architecture: ${TARGETARCH}"; exit 1 ;; \
           esac; \
           RTK_URL="https://github.com/rtk-ai/rtk/releases/download/v${RTK_VERSION}/rtk-${RTK_ARCH}-unknown-linux-musl.tar.gz"; \
           echo "Downloading RTK ${RTK_VERSION} for ${RTK_ARCH}..."; \
           curl -fsSL -o /tmp/rtk.tar.gz "${RTK_URL}"; \
           tar -xzf /tmp/rtk.tar.gz -C /usr/local/bin rtk; \
           chmod +x /usr/local/bin/rtk; \
           rm /tmp/rtk.tar.gz; \
           # Verify installation
           rtk --version
       ```

    3. Ensure tar package is available (it's included in Alpine base, but verify)

    4. Keep the ENTRYPOINT and CMD inherited from runtime-base (already present)
  </action>
